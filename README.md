# ⚔️ Project: Web-Text RPG (Title Pending)

본 저장소는 컴퓨터공학 멘토와 입문자가 함께 구축하는 **웹 환경 기반 텍스트 RPG** 프로젝트입니다. 단순한 기능 구현을 넘어, 데이터 중심의 사고와 객체지향적 설계의 기초를 학습하는 것을 목적으로 합니다.

## 🎯 프로젝트 핵심 목표
1. **데이터와 로직의 분리:** 게임의 상태(State) 데이터와 이를 처리하는 비즈니스 로직을 구분하여 설계합니다.
2. **이벤트 드리븐 설계:** 사용자의 입력(Click, Type)이 프로그램의 상태를 어떻게 변화시키는지 이해합니다.
3. **코드 컨벤션 준수:** 누구나 읽기 쉬운 명명 규칙과 구조를 유지하여 유지보수성을 확보합니다.

## 📅 5-Week Technical Roadmap

### 1주차: 초기화 및 변수 할당 (Initialization)
* **목표:** 게임의 기초가 되는 캐릭터 데이터를 메모리에 할당하고 출력합니다.
* **기술 포인트:** 원시 타입(Primitive Types)과 변수 선언 키워드의 이해.
* **코드 예시:**
```javascript
let playerName = "Adventurer"; 
// let: 블록 스코프(Block Scope)를 가진 변수를 선언하는 키워드로, 이후에 값을 재할당할 수 있습니다.
// playerName: 데이터를 식별하기 위해 메모리 주소에 붙인 이름(식별자)이며, 카멜 케이스(camelCase) 관례를 따릅니다.
// =: 대입 연산자로, 우항의 값을 좌항의 변수에 저장(할당)하는 역할을 수행합니다.
// "Adventurer": 따옴표로 감싸진 문자열(String) 리터럴 데이터로, 캐릭터의 이름을 의미합니다.
// ;: 세미콜론은 자바스크립트에서 하나의 명령문(Statement)이 끝났음을 브라우저 엔진에 알리는 마침표 역할을 합니다.
2주차: 데이터 구조화 (Data Structuring)
목표: 관련 있는 데이터를 객체(Object)와 배열(Array)로 그룹화합니다.

기술 포인트: 참조 타입(Reference Types)과 데이터 접근 방식(Dot notation).

코드 예시:

JavaScript
const player = { hp: 100, atk: 15 };
// const: 한 번 할당하면 참조 주소를 바꿀 수 없는 상수를 선언할 때 사용하며, 객체의 속성 값은 변경 가능합니다.
// { }: 중괄호는 여러 개의 속성(Property)을 키(Key)와 값(Value)의 쌍으로 묶어주는 객체 리터럴 생성 구문입니다.
// hp: 100: 'hp'라는 키에 숫자 100을 할당하며, 객체 내부에서 속성 간의 구분은 쉼표(,)를 사용합니다.
3주차: 흐름 제어 및 모듈화 (Flow Control)
목표: 조건문을 통한 분기 처리와 함수의 캡슐화.

기술 포인트: 비교 연산자 및 함수의 매개변수와 반환값 처리.

코드 예시:

JavaScript
function checkGameOver(hp) { return hp <= 0; }
// function: 특정 기능을 수행하도록 설계된 독립적인 코드 블록인 함수를 정의하는 키워드입니다.
// (hp): 매개변수(Parameter)로, 함수 외부에서 데이터를 전달받기 위해 사용하는 변수입니다.
// return: 함수의 실행을 종료하고, 계산된 결과값을 호출한 곳으로 반환(Return)하는 예약어입니다.
// <=: 비교 연산자로, 좌항이 우항보다 작거나 같은지를 판단하여 참(true) 또는 거짓(false)을 산출합니다.
4주차: 인터페이스 구현 (DOM Manipulation)
목표: JavaScript로 HTML 문서 객체(DOM)를 제어하여 화면을 갱신합니다.

기술 포인트: DOM 선택자와 이벤트 리스너의 바인딩.

5주차: 데이터 영속성 및 최적화 (Persistence)
목표: 브라우저 저장소(LocalStorage)를 활용한 세이브 시스템 구현.

기술 포인트: JSON 직렬화와 로컬 스토리지 API 활용.

🛠 협업 가이드 (Git Strategy)
전문적인 협업을 위해 아래 명령어를 숙달하며, 모든 작업은 개별 커밋으로 기록합니다.

상태 확인: git status (현재 파일들의 변경 및 추적 상태를 확인합니다.)

스테이징: git add . (수정 사항을 다음 커밋에 포함하기 위해 준비 영역에 올립니다.)

기록: git commit -m "feat: 1주차 데이터 정의 완료" (변경 사항에 대한 명확한 요약을 기록합니다.)

전송: git push origin main (로컬의 기록을 원격 저장소로 동기화합니다.)
